| # | カテゴリタイプ | 説明 |
|---|-------------|------|
| A | **業界直接系** | 同業種のYouTubeチャンネル |
| B | **業界投資/分析系** | 業界の市場・投資・データ分析系 |
| C | **関連スキル系** | 業界関連のスキル・ハウツー系 |
| D | **ビジュアル系** | ビジュアル・ルームツアー・ビフォーアフター系 |
| E | **ライフスタイル系** | ターゲット層のライフスタイルVlog |
| F | **マネー・資産系** | お金・節約・資産形成系（10万人以上） |
| G | **専門知識系** | 税金・法律・専門知識系 |
| H | **ビジネス系YouTuber** | ビジネス系大物YouTuber（10万人以上） |

## Phase 5: 企画設計（渾身の10種類 × 動画案各5本）

### 設計ルール

1. **ファネル対応**: 各企画にターゲットファネル層を明記
2. **目的明記**: 認知形成/第一想起/リード獲得 のいずれか
3. **エンタメ性**: 「見てて楽しい」が最優先。教育だけにならない
4. **自分ごと化**: ターゲットが「自分のこと」と感じるタイトル設計
5. **バズ要素**: 少なくとも2企画はショート/バズ狙い

### 出力

## Phase 6: サムネイル生成（Gemini 3 Pro Image）

### 生成方法

```python
import google.genai as genai
from google.genai import types

client = genai.Client(api_key=os.environ.get("GEMINI_API_KEY"))

response = client.models.generate_content(
    model='gemini-3-pro-image-preview',
    contents=prompt,
    config=types.GenerateContentConfig(
        response_modalities=['IMAGE'],
        image_config=types.ImageConfig(
            aspect_ratio='16:9'
        )
    )
)

for part in response.parts:
    if part.inline_data is not None:
        img_data = part.inline_data.data
        break
```

### プロンプト設計ルール

- 「YouTube thumbnail design, 16:9 landscape」から始める
- トーン＆マナーに合わせた雰囲気を指定
- 日本語テキストオーバーレイを含める
- 明るく、エネルギッシュに
- 1企画1枚 = 計10枚

### リトライ

- 画像なしレスポンス→最大3回リトライ（3秒間隔）
- エラー→最大3回リトライ（5秒間隔）
- 各生成間に2秒のインターバル

## Phase 7: PDFポートフォリオ生成

### 構成（18ページ標準）

| ページ | 内容 |
|--------|------|
| 1 | 表紙 |
| 2 | 大方針 × YouTube目的 |
| 3 | ターゲット × ファネル設計 |
| 4 | チャンネルコンセプト × ポジショニング |
| 5 | 100チャンネル分析 × 伸びるパターン |
| 6 | 企画一覧（10種類概要） |
| 7-16 | 企画詳細（1企画1ページ × 10） |
| 17 | サムネイルギャラリー |
| 18 | 裏表紙 |

### 生成方法

1. Pythonで HTML を生成（base64埋め込み画像）
2. Chrome Headless で PDF 変換:
   ```bash
   "/Applications/Google Chrome.app/Contents/MacOS/Google Chrome" \
     --headless --disable-gpu --no-sandbox \
     --print-to-pdf="output.pdf" --print-to-pdf-no-header \
     "input.html"
   ```

### デザインルール

- A4サイズ、日本語フォント（Hiragino Kaku Gothic ProN）
- グラデーション背景ヘッダー
- ファネル別カラーコード使用
- 統計ボックスでKPIを視覚化
- サムネイルは角丸+影付き
- **サムネイル画像は絶対に切らない**: `object-fit: contain; height: auto;` を使う（`max-height + cover` はNG）
- **CSS gradient text はPDFで崩れる**: `-webkit-background-clip: text` は使わず `color:` を使う
- **クライアント送付前にセルフチェック**: 崩れ・切れがないか確認。失礼のないように

## Phase 8: 送付（オプション）

### 外部連絡ルール（必須）

1. **AIパク開示**: メッセージの冒頭で「パクさんの分身のAIパクです！🤖💙」と名乗る
2. **愛を込める**: 絵文字を使い、温かみのあるメッセージにする ✨😊🙏
3. **コミュニケーションを楽しむ**: 事務的にならず、パクさんの「世界を愛する」スタンスを体現

ユーザーが送付先を指定した場合:

### Chatwork

```bash
curl -s -X POST \
  -H "X-ChatWorkToken: $CHATWORK_API_TOKEN" \
  -F "file=@output.pdf;filename=ファイル名.pdf" \
  -F "message=メッセージ本文" \
  "https://api.chatwork.com/v2/rooms/{room_id}/files"
```

**注意**: Chatworkのファイルアップロード上限は5MB。10MB超の場合は画像をJPEG圧縮してから再生成。

### Slack

MCP server-slack の `slack_post_message` を使用。
